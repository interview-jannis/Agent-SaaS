## 2. ⭐ 호텔 가격 × nights

### 배경

호텔 variant base_price가 **객실 1박 가격**으로 들어가 있는데, 견적 만들 때 nights 곱하지 않고 1박 가격 그대로 들어감. 3박 묵어도 1박 비용만 청구되는 셈 — 견적 underbilling.

### 의미 결정

호텔은 다른 상품과 가격 모델이 다름:

| 상품 | 가격 모델 |
|---|---|
| 시술/식사/시설 | `unitPrice × memberCount` (인원만큼 곱) |
| **호텔** | `unitPrice × nights` (객실당, 인원 무시) |

(b)로 결정. 객실 자체 비용이라 2명이 1실 공유해도 1실 가격. 만약 agent가 "2명 그룹 → 객실 2개" 잡으려면 호텔 variant를 2번 추가해야 함.

### 구현

[`lib/pricing.ts`](src/lib/pricing.ts) 신규 헬퍼:
```ts
export function isHotelItem(category, subcategory): boolean {
  return category === 'Subpackage' && subcategory === 'Hotel'
}
export function nightsBetween(dateStart, dateEnd): number {
  // 둘 중 하나 missing이거나 end <= start면 1로 fallback
}
```

[`agent/home/page.tsx`](src/app/agent/home/page.tsx) cart 총액:
- `nights = nightsBetween(dateStart, dateEnd)` useMemo
- 라인별 multiplier: 호텔이면 nights, 그 외 memberCount

[`agent/home/review/page.tsx`](src/app/agent/home/review/page.tsx):
- `itemMultiplier(item, group)` 헬퍼 — 호텔/비호텔 분기
- `groupSubtotalUSD` 호출에 multiplier 적용
- 표 행: `× 3 nights` (호텔) / `×2` (인원) 분기 라벨
- `handleSendQuote` `addDocumentItem`에 multiplier 곱한 base/final 저장
- **variant_label_snapshot에 ` · 3 nights` 베이크** — 견적 만든 시점의 nights를 라벨에 박아넣어 finalize 후 case dates 바뀌어도 견적 라벨은 유지
- `quantity` 컬럼에도 multiplier 저장 (DB 진실)

### 자동 반영되는 곳

[`QuoteDocument.tsx`](src/components/QuoteDocument.tsx) / [`SelectedProductsSection.tsx`](src/components/SelectedProductsSection.tsx):
- `final_price`에 nights 이미 곱해져 들어감 → 그대로 렌더 시 정확한 라인 총액
- `variant_label_snapshot`에 `· 3 nights` 자동 노출
- → **이 두 컴포넌트 코드 수정 0줄**. 데이터만 정확히 들어가면 UI 자동 반영되는 게 변수-스냅샷 모델의 가치.

## 4. Variant 모달 가격 desc 정렬

문제: SOFITEL 호텔 detail 모달에서 객실 옵션이 알파벳/sort_order 순. `Luxury Lake Room ($243)` → `Prestige Suite ($473)` → `Opera Suite ($1,014)` → `Presidential Suite ($14,884)` → `1-Bedroom Premier ($338)`. 가격 차이 200배인데 시각적 우선순위 없음.

VIP 카탈로그는 비싼 옵션이 먼저 노출돼야 검토 동선이 자연스러움 (4/30 결정한 카드 정렬 정책의 모달 버전).

[`agent/home/page.tsx`](src/app/agent/home/page.tsx) detail 모달 variants:
- 기존: `.sort((a,b) => a.sort_order - b.sort_order)`
- 변경: `.sort((a,b) => variantUsd(b) - variantUsd(a) || a.sort_order - b.sort_order)`
- `variantUsd` 클로저를 sort 위로 끌어올림 (이전엔 sort 후 정의)
- tie-break은 sort_order 유지

2-level 아코디언 그룹핑(`· ` separator)은 이미 정렬 끝난 variants 위에서 동작하므로 그룹 내부도 자동 가격 desc.
