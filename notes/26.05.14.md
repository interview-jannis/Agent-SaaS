### 6. 세그먼트 컬럼 레이아웃 (그룹별 분리, 16:29)

**문제:** 4명 케이스, Group 1(남 2명)은 Asan, Group 2(여 2명)는 Samsung. flat list로 늘어놓으면 두 그룹이 같은 날 어떻게 다른 시술을 받는지 한눈에 안 보임.

**해결: Shared 앵커 기준 세그먼트 분할.**
- 그룹 있을 때:
  - Shared 아이템(`groupId === null`) = 풀-width 행
  - 그룹 아이템 = 그룹별 컬럼 분리 (예: Group 1 | Group 2 | Group 3)
  - Shared 앵커를 기준으로 세그먼트 자르고, 각 세그먼트의 컬럼은 `justify-evenly`로 정렬
- 그룹 없을 때 (단일 그룹 또는 그룹 미지정): 기존 flat list 그대로
- 상단에 **그룹명 헤더 행** 추가 — 어느 컬럼이 어느 그룹인지 명시

### 10. Admin-side Bug Sweep (17:44)

하루 작업 마무리 정리 라운드.

- **`fillTransfers`**:
  - pending 아이템도 detection에 포함 (저장 전 transfer도 중복 방지)
  - pending transfer가 이미 있으면 중복 insert 차단
  - renumber pass가 day 전체 커버 → `sortOrder` 일관성
- **`applyVariantPick`**:
  - deselect 시 / 비-hotel 상품으로 전환 시 `hotelCheckType` 클린업
  - auto-filled title("Hotel Check-in/Out/Stay") 인식 → 해당 title도 클린업
- **Segmented layout**: 비연속 멀티그룹 아이템(예: Group 1+3에만 적용)을 컬럼 중복 셀로 그리지 않고 **풀-width emerald 서브셋 배너**로 렌더. "Covers G1 · G3" chip으로 명시.
- **`moveItem`**: peer 정렬을 `compareScheduleItems`로 통일 → ▲/▼ 버튼이 디스플레이 순서와 일치
- **`commitPendingItem`**: items를 ref로 읽어서 최신 키스트로크가 저장 draft에 반영됨 (stale state bug)
- **`nextSortOrderForDay` 헬퍼**: `max(sortOrder)+1`, marked-for-removal 제외. `addItem/addPrayerTime/addFreeTime/createTransferFromSelection/createHotelBookendFromSelection` 5곳 통일.

## 결정 사항

1. **Hotel = appointment 한 종류** — 별도 itemType 아님. product picker에서 hotel 카테고리 상품 선택 → 서브드롭다운으로 Check-in/out/Stay 결정.
2. **Trip Services = appointment 옆 별도 picker** — interpreter/concierge/security는 product picker에서 빠지고 복수 pill로 분리.
3. **케이스 페이지 레이아웃 = mixed-width** — schedule 섹션만 풀-width, 나머지는 narrow editorial.
4. **Sticky는 schedule 안에선 bottom bar만** — 헤더 sticky는 모두 제거 (스크롤 방해).
