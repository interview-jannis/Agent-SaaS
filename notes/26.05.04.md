## 5. ⭐ Product Variants 모델 도입 (큰 작업)

### 배경
같은 시술의 다른 회차/사이즈를 별도 product로 만드는 건 카탈로그 bloat. DIAR Membership Red/Purple/Black, MCT Stem Cell 1/3/5 sessions, Thermage 300/600/900 shots 같은 케이스. agent 카탈로그에선 1 카드만 보이고 디테일에서 variant 선택하는 게 자연스러움.

### 스키마 — `sql/2026-05-04_product_variants.sql`
```sql
CREATE TABLE product_variants (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  product_id UUID NOT NULL REFERENCES products(id) ON DELETE CASCADE,
  variant_label TEXT,           -- NULL = 단일 variant (default)
  base_price NUMERIC NOT NULL,
  price_currency TEXT NOT NULL DEFAULT 'KRW',
  sort_order INT DEFAULT 0,
  is_active BOOLEAN DEFAULT TRUE,
  created_at TIMESTAMPTZ DEFAULT now()
);
ALTER TABLE document_items
  ADD COLUMN variant_id UUID REFERENCES product_variants(id) ON DELETE SET NULL,
  ADD COLUMN variant_label_snapshot TEXT;
```

`products.base_price`는 representative price로 보존 (compat). 신규 코드는 variants 테이블에서 읽음. 기존 documents는 default variant id로 backfill.

### Pricing 룰 — `lib/pricing.ts`
4/30 SOP 결정에 따른 카테고리별 마진 룰:
- **마진 적용**: K-Medical, K-Beauty, K-Wellness > **Spa만**, K-Starcation, K-Education
- **원가 그대로**: K-Wellness > Tour/Leisure/Shopping/K-Content, Subpackage 전체

```ts
export function appliesMargin(category, subcategory): boolean {
  if (category === 'K-Wellness') return subcategory === 'Spa'
  return ['K-Medical','K-Beauty','K-Education','K-Starcation'].includes(category)
}
export function variantPriceUsd({basePrice, priceCurrency, exchangeRate, marginMult, applyMargin}): number
export function variantPriceKrw(...): number
```

### Agent home cart 마이그레이션
- `Group.productIds: string[]` → `Group.items: { productId, variantId }[]`
- localStorage 마이그레이션 (옛 cart 형태면 자동 폐기)
- Single variant: + Group 1로 즉시 추가. Multi-variant: detail 모달에서 라디오로 선택 (variant마다 가격 + tap to add)

### Review page 마이그레이션

- `addDocumentItem` 호출 시 `variantId` + `variantLabelSnapshot` 추가 (snapshot이 있으면 finalize 후 variant 삭제돼도 견적엔 라벨 보존)

### Quote/Invoice rendering

## 6. ⭐ Excel 일괄 업로드 (UPSERT + dry-run preview)

엑셀 마스터 파일을 admin이 직접 올려서 product/variant 일괄 등록·갱신.

### API — `/api/admin/products/upload-excel`
- multipart xlsx 받아서 SheetJS로 파싱
- Group by `(category, partner_name, name)` → product family. 같은 그룹 내 row들은 variant로 묶음
- **Match key**: `(category, partner_name, name)` natural key
- **차이 판별**: existing row와 desired row 비교 → 동일 skip / 다름 update / 신규 insert
- Variant: variant_label로 매칭, base_price/currency/sort_order/is_active diff
- **Subcategory**: 신규면 자동 생성 (`product_subcategories`에 INSERT)
- **Category**: 신규면 무시 (의도적 — 카테고리는 6개 fixed, Manage Categories에서 명시 추가만 허용)
- Solo 그룹 (variant 1개)은 variant_label NULL로 정리 (excel에서 잘못 들어간 라벨 자동 청소)

### Dry-run preview
`?dryRun=true` 쿼리로 호출 시 모든 write 스킵하고 카운트만 반환. Updated 항목은 field-level diff 함께.

### UI 플로우 (admin/products 페이지)
1. **Upload Excel** 버튼 (Add Product 옆, 같은 primary green) → 파일 선택
2. 자동 dry-run → **Review changes 모달** 노출:
   - Products / Variants 카운트 (신규/업데이트/그대로)
   - **Updated 표** (Type / Item / Field / Before / After) — sticky header, 같은 item의 여러 field는 rowspan으로 묶음
   - **Missing notice** — DB엔 있지만 이번 엑셀엔 없는 항목 카운트 + 펼침 list ("delete via product detail" 안내, 자동 삭제 안 함)
   - Errors / notices
3. **Confirm Save** / **Cancel**
4. Confirm 시 동일 파일로 두 번째 호출 (write 적용) → 결과 banner

### Safety
- 동일 파일 다시 올려도 unchanged로 카운트만 늘고 DB는 그대로
- 부분 업로드 (예: K-Beauty만) 해도 다른 카테고리 product 안 지움
- 의도적 삭제는 product detail 페이지에서
