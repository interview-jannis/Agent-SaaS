### 30. **`can_sign_contracts` 권한 분리** (d81f12f)
배경: 계약서 사인은 대표님(법인 representative)만 가능해야 하는데 SaaS 운영은 다른 admin이 맡음. 기존엔 `is_super_admin` 하나로 모든 권한 묶여있어서 super admin이 곧 사인 가능.

- 신규 컬럼: `admins.can_sign_contracts BOOLEAN DEFAULT false`
- 마이그레이션: 기존 `is_super_admin = true`는 `can_sign_contracts = true`로 백필 (안전 fallback)
- 권한 매트릭스:
  | 역할 | is_super_admin | can_sign_contracts |
  |---|---|---|
  | 대표님 | ✓ | ✓ |
  | 운영 manager (본인) | ✓ | ✗ |
  | 일반 admin | ✗ | ✗ |
- 게이트 적용:
  - `/api/admin/sign-contract` (agent contracts): `can_sign_contracts` 없으면 403
  - `/api/case-contracts/sign-admin` (3-party contracts): 동일
  - `/admin/agents/[id]/contract/[contractId]` UI: 권한 없으면 amber 안내 + 사인 버튼 숨김
  - `AdminCaseContractSection`: 권한 없으면 `canAdminSign = false` + amber 안내
- `/admin/admins` UI: **Edit 모달 도입** — name/title/is_super_admin/can_sign_contracts 한 번에 편집. 마지막 super admin 제거 lockout 가드

### 32. `blockFromTime` 비대칭 boundary 규칙
- 기존: 06:00/12:00/18:00 boundary가 항상 이전 블록에 속함 (`18:00` 시작도 afternoon)
- 직관 어긋남: "18:00 시작" = 저녁 식사 시작 → evening 느낌인데 afternoon으로 분류
- 신규 규칙 (`types/schedule.ts`):
  - `mode='start'` (기본): 06:00 → morning, 12:00 → afternoon, 18:00 → evening (boundary가 **다음** 블록)
  - `mode='end'`: 06:00 → night, 12:00 → morning, 18:00 → afternoon (boundary가 **이전** 블록)
- 호출부:
  - ScheduleEditor 시작 시간 입력: default `'start'`
  - ScheduleEditor 끝 시간 입력 (endTime → block / endBlock): `'end'`
  - ScheduleDocument `segEndBlock`: segStart는 `'start'`, segEnd는 `'end'`

  ```sql
  SET total_price = (
    SELECT COALESCE(SUM(di.final_price), 0)
    FROM document_groups dg JOIN document_items di ON di.document_group_id = dg.id
    WHERE dg.document_id = d.id AND di.removed_at IS NULL
  )
  FROM cases c
  WHERE d.case_id = c.id AND c.case_number = '#C-001'
    AND d.type = 'final_invoice';
  ```
