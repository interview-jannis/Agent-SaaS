## 3. Admin Edit Selected Products — Trip Services 수량/초과근무 인라인 조작

### Trip Services days/nights 조작
- 각 Trip Services 아이템 행에 `−` / `Xn` 또는 `Xd` / `+` 인라인 컨트롤 추가
- 호텔 아이템: `n`(nights) 단위, 나머지: `d`(days) 단위
- `updateItemQuantity(itemId, currentQty, currentFinalPrice, newQty)`: 단가 역산 후 새 수량 곱해 `final_price` 재계산 → DB 즉시 반영

### Overtime hours 컨트롤 (호텔 제외 Trip Services)
- amber 색상 `−` / `+Xh` / `+` 컨트롤로 초과근무 시간 수동 보정
- 시간당 단가(`overtime_rate_krw`)는 `product_variants` 에서 조회
- KRW 추가금 실시간 표시: `base_price×qty + ot×otRate`
- `updateItemOvertimeHours(item, newOT)`: `overtime_hours` + `final_price` 동시 업데이트
- `calcAndStoreOvertimeHours()`: Save Draft 시 `day_subpackages` JSONB에서 실제 투입 시간 계산 → DB 자동 기록
  - 계약 시간 = `quantity × duration_value`(hour 단위 상품만), 초과 = max(0, 실제 - 계약)

### Edit Selected Products — finalInvoice 우선 표시
- `docList`를 `finalInvoice ?? latestQuote` 기준으로 구성하도록 변경
- quotation은 불변(immutable) — Edit Selected Products 후 항목 추가/제거는 final_invoice에 반영되므로, finalInvoice가 있으면 그걸 기준으로 표시

## 10. Subpackage markup=0 견적 가격 버그 수정 + 차액 로직

### 버그 원인
`system_settings`에 `markup_rates` row가 없거나 `Subpackage-Other` 키가 누락된 경우, review 페이지가 `DEFAULT_MARKUP_RATES`(0.3)로 fallback → 마크업이 적용된 `final_price` 저장.

### 수정 1 — 제출 시 DB 재조회 (review/page.tsx)
`handleSendQuote` 내부에서 `markup_rates`를 DB에서 직접 재조회 (`liveMarkupRates`) → 페이지 로드 시점 state 의존 제거.

### 수정 2 — Subpackage Free = 최저 variant 무료 + 차액만 청구
비즈니스 룰: Subpackage markup=0은 "무료 포함" 의미. 최저가 variant = ₩0, 상위 variant = (선택가 − 최저가) 차액만 청구.

```
final_price = max(0, 선택 variant KRW − 해당 상품 최저 variant KRW)
```

적용 위치:
- `review/page.tsx`: `subpkgUpgradeKrw()` 헬퍼 추가. `itemUsd` / `serviceItemUsd` / `groupsKRW` / `servicesKRW` / `finalUnitKrw` (group + trip services) 전부 반영
- `product/page.tsx`: `subpkgUpgradeUsd()` 헬퍼 추가. `totalUSD` useMemo + group item 표시 + trip service 표시 + detail 모달 `variantUsd` 계산 수정 (이전엔 모든 variant를 0으로 반환)
