# 연구노트 — 2026.04.20

## 작업 개요

Agent SaaS MVP 1주차 후반. Admin / Agent 양쪽 UI를 대폭 개편한 날.

### 3. Agent > Cases > [id] — 신규 파일 생성

`/agent/cases/[id]/page.tsx` 새로 만듦.

**표시 내용**:
- 상단 바: `← Cases / #C-xxx` 브레드크럼 + 상태 배지
- Travel Period (편집 가능, 날짜 인풋)
- Lead Client (이름 클릭 → `/agent/clients/[id]` 링크)

- Financials (총액 USD, 결제 마감일, 에이전트 예상 수익)

**에이전트 수익 계산식**:
```
earnings = total_price × agent_margin_rate / (1 + agent_margin_rate)
```

**기술 이슈**: Next.js 15+ App Router에서 `params`는 `Promise`임.
클라이언트 컴포넌트에서는 `useParams<{ id: string }>()` 사용.

### 6. Agent Home — 동일 상품 멀티 그룹 허용

**기존 문제**: `toggleProduct` 함수가 A그룹에 상품을 추가할 때 다른 그룹에서 해당 상품을 자동 제거하고 있었음.

**원인**: 
```js
// 문제 코드
return { ...g, productIds: g.productIds.filter((id) => id !== productId) }
```

**수정**:
```js
// 수정 후 — 다른 그룹은 그대로 둠
return g
```

이제 Group A와 Group B가 동일한 상품을 각각 선택할 수 있음.

**카드 테두리 표시도 수정**: 현재 활성 그룹에 해당 상품이 있을 때만 그 그룹 컬러로 테두리 표시.

### 7. Admin > Cases — 표 형식 + 50/50 분할 뷰로 전면 개편

**기존 문제**: 좌측 카드 리스트(w-80) + 우측 고정 너비 패널. 사용자 피드백으로 두 번 width 조정 왔다갔다.

**최종 구조**:

**케이스 미선택 시** → 전체 너비 테이블
- 컬럼: 케이스# / 에이전트 / Lead Client / 상태 / 인원 / 여행 기간 / 총액(USD)
- 상단 상태 필터 드롭다운

**케이스 선택 시** → 상단 바 (`← Cases / #C-xxx`) + 하단 50/50 분할
- **왼쪽 50%**: 에이전트 정보, 여행 기간, Lead Client 상세, 동반자, Financials(KRW+USD 둘 다), Schedule
- **오른쪽 50%**: Selected Products (그룹별 상품, USD+KRW 표시, 소계, 총계)

**Admin Actions** (왼쪽 하단):
- `payment_pending` → 결제일 입력 + "Confirm Payment" 버튼
- `payment_completed` / `schedule_reviewed` → PDF 업로드 버튼
- `schedule_confirmed` → "Mark Travel Complete" 버튼

**"View Invoice" 링크 제거** — Admin은 인보이스를 고객에게 보내는 역할이 아니므로 불필요. 대신 Financials 섹션 우측에 조용한 `View ↗` 링크만 남김.

**개념 정리**:
- Quote = Invoice. `/quote/[slug]`가 인보이스 페이지임.
- 별도 Invoice 발행 단계는 없음. 견적 생성 = 인보이스 생성.

- 변경: "Preview ↗" (눈 아이콘, 에이전트가 직접 확인용) + "Send Invoice" 버튼 (클립보드에 URL 복사, 2초간 "Copied!" 표시)

- Financials 헤더에 `View ↗` 소형 링크만 유지 (확인용)

### 9. Invoice 페이지 404 버그 수정

**원인**: `supabase-server.ts`에서 `SUPABASE_SERVICE_ROLE_KEY!`를 사용 중이었는데, `.env.local`에 해당 환경변수가 없으면 Supabase 클라이언트 초기화 자체가 실패함. 결과적으로 쿼리가 항상 null을 반환하고 `notFound()`가 호출됨.

**수정**:
```ts
// 수정 전
process.env.SUPABASE_SERVICE_ROLE_KEY!

// 수정 후
process.env.SUPABASE_SERVICE_ROLE_KEY ?? process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY!
```

RLS가 전체 비활성화돼 있으므로 anon key로도 모든 쿼리 가능.

**주의**: 수정 후 반드시 개발 서버 재시작 필요 (`Ctrl+C` → `npm run dev`).

## 발생했던 주요 이슈 & 해결

| 이슈 | 원인 | 해결 |
|------|------|------|
| Agent 사이드바에 Cases 없음 | NAV 배열에 추가 안 됨 | Cases 항목 추가 |
| Invoice 누르면 404 | `SUPABASE_SERVICE_ROLE_KEY` 미설정 | anon key fallback 추가 |
| 금액이 원화로 표시됨 | 에이전트 화면에 KRW 그대로 노출 | 전부 USD 환산으로 변경 |
| 동일 상품을 두 그룹에 못 씀 | `toggleProduct`가 다른 그룹에서 자동 제거 | 다른 그룹 건드리지 않도록 수정 |
| Client 편집에 여행 필드 포함 | 설계 오류 | 여행 필드는 Case에만 포함, Client에서 제거 |
| Admin Cases 레이아웃 왔다갔다 | width 값 조금씩 수정 반복 | 구조 자체를 표→50/50으로 전면 재설계 |
| `className="input"` 정의 안 됨 | Tailwind에 없는 커스텀 클래스 사용 | 명시적 Tailwind 클래스로 교체 |

## 느낀 점 / 메모

- **UI 패턴 통일이 핵심**: 표→상세 패턴을 명확히 정하고 나니 설계가 훨씬 빨라짐. 초반에 카드+패널 혼재하던 게 계속 혼란을 줬음.
- **환경변수 문제가 무서운 이유**: 런타임 에러가 아니라 `notFound()`로 조용히 실패하니 원인 파악이 어려웠음. 서버 컴포넌트는 특히 주의.
- **Invoice = Quote 개념 통일**: 별도 Invoice 발행 단계를 만들려다가 불필요함을 확인. 견적 생성 시점에 slug 발급 → 그게 인보이스 URL. 심플하게 유지.
- **MVP 원칙**: 안 쓰이는 탭보다 핵심 플로우(상품선택→견적→결제확인→스케줄)가 완벽하게 동작하는 게 우선.
