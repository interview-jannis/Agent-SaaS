## 1. 프로젝트 개요 확정

**비즈니스 플로우**

```
1. Agent가 고객 등록 + 상품 선택 → 견적 생성
2. 시스템이 고객 이메일로 결제 요청 자동 발송
3. 결제 완료 → Admin이 수동 확인 처리
4. Admin이 스케줄 PDF 업로드 → URL 자동 생성
5. Agent와 고객이 스케줄 확인 후 확정 또는 재요청
6. 여행 완료 → Admin이 수동 처리 → 마진율 자동 재계산
```

**Admin 수동 개입 포인트 (3회)**

- 결제 완료 확인 (실제 입금일 수동 입력 + 확인 처리일 자동 기록)
- 스케줄 PDF 업로드
- 여행 완료 처리

**기술 스택**

- DB: Supabase (PostgreSQL)
- 인증: Supabase Auth (Agent/Admin 역할 분리)
- 파일 저장: Supabase Storage (스케줄 PDF, 상품 이미지)
- 이메일: Resend (결제 요청 자동 발송)
- 배포: Vercel

## 2. 가격 및 마진율 구조 확정

**견적가 계산 수식**

```
고객 견적가 = 원가 × (1 + 회사 마진율) × (1 + 에이전트 마진율)
예시: 100만원 × 1.5 × 1.15 = 172.5만원
```

**저장 방식**

- 상품: 원가만 저장
- 회사 마진율: system_settings 전역 설정값
- 에이전트 마진율: agents 테이블에 귀속, 자동 계산

**에이전트 마진율 자동 산정**

- 기준: 해당 월 "여행 완료" 처리된 고객 수
- 0~10명: 15% / 11~30명: 20% / 31명+: 25%
- Admin이 여행 완료 처리 시 자동 재계산, 월 단위 리셋

## 6. 화면 구조 확정

**로그인**

- 단일 로그인 화면 → admins/agents 테이블 조회 후 역할별 자동 분기

**Agent (5탭)**

- Home: 상품 탐색 + 장바구니 + 견적 생성
- Clients: 고객 목록 + 고객 상세 + 케이스 이력
- Payouts: 커미션 현황 + 정산 요청
- Dashboard: 케이스 현황 요약 + 마진율 현황
- Profile: 개인정보 + 정산 정보 + 비밀번호 변경

**Admin (5탭)**

- Overview: 액션 필요 케이스 + 매출 현황 + Top 에이전트
- Cases: 케이스 목록 + 단계별 상세 + 스케줄 업로드
- Products: 상품 목록 + 등록/수정
- Agents: 에이전트 목록 + 상태 관리 (읽기 전용 + 활성화 설정)
- Settlement: 정산 요청 확인 + 지급 처리

**고객용 URL (로그인 없음)**

- `/quote/{slug}`: 견적서 확인 + 입금 정보
- `/schedule/{slug}`: 스케줄 PDF 뷰어 + 확정/재요청 버튼

## 7. Agent Home 탭 핵심 설계

**상품 탐색 필터 구조**

- 카테고리 (Medical / Beauty / Tour / K-Starcation / K-Education / Sub-Package)
- 무슬림 친화: 기도공간 유 / 식이 제한 (4종) / 여성 의료진 필수

**장바구니 + 그룹 구조**

- 기본값: 전체 상품이 메인 그룹에 자동 배정
- [+ 그룹 추가] 버튼으로 새 그룹 생성, 그룹명 직접 입력
- 상품 카드에 그룹 색 테두리로 소속 그룹 표시
- 하단 고정 바: 그룹 탭 + 총합 + [견적 만들기]

**견적 생성 분기**

- 등록 고객: 상단에서 먼저 선택 → 상품 탐색 → 견적 만들기
- 미등록 고객: 상품 탐색 → 견적 만들기 → 고객 등록 팝업

**견적 검토 화면**

- 고객 정보 + 동반자 추가 + 그룹 멤버 배정 + 견적 내용 확인
- [수정하기] / [견적서 발송] 버튼
