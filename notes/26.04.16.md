## 2. 인증 (Auth)

### 로그인 페이지 (`/login`)
- Supabase Auth 이메일/비밀번호 로그인
- 로그인 성공 후 `admins`, `agents` 테이블을 `Promise.all`로 동시 조회
- 역할 분기:
  - `admins.auth_user_id` 일치 → `/admin/overview`
  - `agents.auth_user_id` 일치 → `/agent/home`
  - 둘 다 없으면 → signOut + 에러 메시지
- `useSearchParams` → Suspense 래핑 (빌드 오류 해결)
- 가입 완료 후 `/login?registered=1` 리다이렉트 → 딥 그린 성공 배너 표시
- 디자인: 흰 배경, 딥 그린(`#0f4c35`) 포인트, 미니멀

### 회원가입 페이지 (`/register`)
- Agent 전용 가입 (이름/이메일/비밀번호/연락처/국가)
- 플로우: `auth.signUp()` → `generate_agent_number()` RPC 호출 → `agents` 테이블 insert
- `is_active: true` — 가입 즉시 로그인 가능 (Admin 승인 불필요, MVP 결정)
- 완료 후 signOut → `/login?registered=1` 리다이렉트

### RLS 이슈 해결
- 초기: `admins`, `agents` 테이블 SELECT 정책 없어 쿼리가 null 반환
- 해결: SQL Editor에서 `auth.uid() = auth_user_id` 조건 SELECT 정책 추가
- 이후: 전체 테이블 RLS 비활성화 결정 (내부 B2B 도구, 외부 접근 없음)

## 3. Admin 화면

### 공통 레이아웃 (`src/app/admin/layout.tsx`)
- 서버 컴포넌트 레이아웃 + 클라이언트 사이드바 분리
- `SessionGuard` 컴포넌트 포함

### AdminSidebar (`src/components/admin/AdminSidebar.tsx`)
- `usePathname()`으로 active 항목 딥 그린 하이라이트
- `getSession()`으로 로그인 유저 이름 DB 조회 표시 (React Strict Mode navigator lock 오류 방지)
- 로그아웃 버튼 (`signOut()` → `/login`)
- 미구현 탭 (Agents, Settlement) 비활성화 처리 (`disabled` 플래그)
- 흰 배경 + 연한 회색(`bg-gray-50`) 사이드바

**네비게이션:** Overview / Cases / Products / Agents / Settlement / Settings

### Admin Overview (`/admin/overview`)
- `export const dynamic = 'force-dynamic'` 필수 (정적 렌더링 404 방지)
- 6개 쿼리 `Promise.all` 병렬 실행

| 섹션 | 데이터 소스 |
|------|-----------|
| Awaiting Payment Confirmation | `cases` where `status = payment_pending` |
| Schedule Upload Required | `cases` where `status = payment_completed` |
| Total Revenue (this month) | `travel_completed` cases의 `quotes.total_price` 합산 |
| Completed Cases / New Clients | 이번 달 `created_at` 기준 count |
| Top Agents This Month | 완료 케이스 agent별 집계, revenue 내림차순 |
| Recent Cases | 최신 5건 |

## 5. DB 변경사항

```sql
-- price_currency 컬럼 추가
ALTER TABLE products ADD COLUMN IF NOT EXISTS price_currency TEXT NOT NULL DEFAULT 'KRW';

-- 전체 테이블 RLS 비활성화 (내부 B2B 도구)
ALTER TABLE products DISABLE ROW LEVEL SECURITY;
ALTER TABLE product_categories DISABLE ROW LEVEL SECURITY;
ALTER TABLE product_images DISABLE ROW LEVEL SECURITY;
ALTER TABLE agents DISABLE ROW LEVEL SECURITY;
ALTER TABLE admins DISABLE ROW LEVEL SECURITY;
ALTER TABLE clients DISABLE ROW LEVEL SECURITY;
ALTER TABLE cases DISABLE ROW LEVEL SECURITY;
ALTER TABLE case_members DISABLE ROW LEVEL SECURITY;
ALTER TABLE quotes DISABLE ROW LEVEL SECURITY;
ALTER TABLE quote_groups DISABLE ROW LEVEL SECURITY;
ALTER TABLE quote_group_members DISABLE ROW LEVEL SECURITY;
ALTER TABLE quote_items DISABLE ROW LEVEL SECURITY;
ALTER TABLE system_settings DISABLE ROW LEVEL SECURITY;

-- generate_agent_number RPC 함수
CREATE OR REPLACE FUNCTION generate_agent_number()
RETURNS TEXT AS $$
DECLARE
  next_num INT;
BEGIN
  SELECT COUNT(*) + 1 INTO next_num FROM agents;
  RETURN '#AG-' || LPAD(next_num::TEXT, 3, '0');
END;
$$ LANGUAGE plpgsql;
```

## 7. 해결한 빌드 오류 / 트러블슈팅

| 오류 | 원인 | 해결 |
|------|------|------|
| Turbopack FATAL panic | USB 연결 시 F드라이브로 경로 캐시됨 | `.next` 삭제 후 E드라이브에서 재시작 |
| `useSearchParams` 빌드 실패 | Suspense 미적용 | login 페이지 Suspense 래핑 |
| TypeScript 타입 오류 | Supabase 중첩 select 타입 추론 | `as unknown as T` 캐스트 |
| `params` 타입 오류 | Next.js 15+ breaking change (Promise) | `await params` 적용 |
| Admin Overview 404 | `force-dynamic` 누락으로 정적 렌더링 | `export const dynamic = 'force-dynamic'` 추가 |
| "Access denied" 로그인 실패 | admins 테이블에 데이터 있어도 RLS가 SELECT 차단 | RLS 전체 비활성화 |
| `NavigatorLockAcquireTimeoutError` | React Strict Mode에서 `getUser()` navigator lock 경합 | `getSession()`으로 변경 |
| "User already registered" | 첫 가입 실패 시 `auth.users`에 고아 레코드 생성 | Supabase 대시보드에서 수동 삭제 |
