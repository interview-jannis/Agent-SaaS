## 8. ScheduleEditor 대수술

### Free time 드롭다운

Day 카드 헤더의 "+ Add Item" 옆에 `+ Free time ▾`:
- Morning free / Afternoon free / Evening free / Full day free
- Full day는 morning+afternoon+evening 3 row 일괄 추가
- 모두 title="Free time" 자동 설정 → 저장 검증 통과

### Add Day / Remove Day 제거

여행 기간이 travel dates에서 파생되니 schedule editor에서 day 추가/삭제할
의미가 없음 (불일치 발생). 제거하고 카피 정리.

### 데이터 모델 확장 (JSONB라 SQL X)

[src/types/schedule.ts](src/types/schedule.ts) ScheduleItem:
```ts
endBlock?: ScheduleItemBlock | null  // null = 단일 block
endTime?: string | null              // 종료 시각
partner?: string | null              // partner brand (eyebrow)
internalNotes?: string | null        // 콘시어지 전용
groupId?: string | null              // document_groups.id; null = Shared
```

### Block / Time 범위

블록 → end block (Same/Morning/Afternoon/Evening) → 시작 시각 → 종료 시각.
ScheduleDocument에서 `09:00 – 15:00 · Morning → Afternoon` 같이 범위 표현.
사용자가 "정확한 시간 모를 수도" 지적 — block 범위만으로도 의미 전달 가능.

### Time24Input — locale-independent 24h

native `<input type="time">`이 OS/브라우저 locale에 따라 12h AM/PM으로 뜨는
이슈 발견 (en-US locale). HTML cross-browser로 24h 강제 옵션 없음.
`<select>` 두 개 (HH/MM)도 이전 라운드에서 작은 드롭다운 스크롤 빠지는 문제로
거부됐음. 결국 `<input type="number">` 두 개로 안착:
- HH (0-23) + MM (0-59), inputMode=numeric
- 타이핑 / 화살표 키 / 호버 스피너로 입력
- WebKit 스피너 CSS 숨김
- onBlur 시 0-padding + clamp

DateTime24Picker (flight 시간) + ScheduleEditor 양쪽에 적용.

### Partner eyebrow 분리

`Gil Hospital · VIP Premium · Female` 한 줄에 다 박힌 title이 시각적으로
부담. partner를 분리 필드로 빼고 작은 uppercase 라벨로 노출:
```
GIL HOSPITAL    (작은 회색 eyebrow)
VIP Premium · Female  (큰 serif title)
```
applyVariantPick은 `partner = cp.partnerName`, `title = ProductName · Variant`
로 분리 저장. deselect 시 admin이 손대지 않은 자동 채움 값만 clear (free-form
text는 보존).

### Internal note (admin only)

"콘시어지 핸드오버" 정보(드라이버, 컨택트, prep)는 운영용. 이전엔 `notes`
하나에 다 박혔는데, VIP 출력에 보이면 부적절. 신규 `internalNotes` 필드:
- Editor: gray-50 dashed-border input "Internal note (driver, contact, prep…)"
- Location도 internal로 demote (풀 주소는 VIP 입장에서 노이즈, 운영용)
- ScheduleDocument: `showInternalNotes=true` 모드에서만 amber 박스로 노출
- /schedule/[slug]?internal=1 쿼리로 admin preview만 internal 보임
- Admin preview 링크에 `internal=1` 자동 부착 (Schedule History "Preview"
  + Build Schedule 헤더 "Preview last version" 둘 다)
- Agent의 Copy/Send 링크는 그대로 (no internal)

### Confirmed 후 internal note 인라인 편집

스케줄 컨펌된 이후에도 콘시어지 정보(드라이버, 컨택트)는 운영 단계에서 변경
가능성 ↑. title/time/block 다 잠그되 internalNotes만 in-place 편집:
- Schedule History의 latest+confirmed 카드에 "Internal Notes (concierge
  handover)" 섹션
- Edit 버튼 → 인라인 편집 (각 row title 표시 read-only, internalNotes input만)
- Save: schedules.items JSONB UPDATE in-place (새 version 생성 X — 컨펌 사실
  자체는 변하지 않음)
- audit log `schedule.internal_notes_updated`

### Group 분리 (단일 schedule + URL 필터)

각 그룹(Female/Male)이 다른 시술 받는 케이스 대응. 옵션 A(그룹별 별도 schedule)
아니라 옵션 B 채택: 단일 schedule + 항목별 group 태그 + URL 필터.

- ScheduleItem.groupId = document_groups.id (null = Shared)
- ScheduleEditor row에 Group 셀렉트 (caseGroups.length > 1일 때만 노출)
- ScheduleDocument.filterGroupId prop: 있으면 `groupId === filter || !groupId`만
  렌더 (VIP 그룹별 뷰), 없으면 모두 + 그룹 배지 (admin 풀 뷰)
- /schedule/[slug]?group={groupId} 쿼리로 그룹 필터
- Editor에 그룹별 좌측 색띠 (4 토너 순환: pink/sky/violet/amber + Shared=gray)
  + 셀렉트 자체를 colored chip 스타일

### Pending row 상태 + 인라인 Cancel/Save row

addItem이 row 추가 + pendingItemIds에 등록 → 노란 점선 박스 + amber 배경.
이동/삭제 버튼 비노출, 행 하단에 "Draft" 안내 + Cancel + Save row 버튼.
- Save row: title 비어있으면 disabled, 클릭 시 pending 해제
- Cancel: row 제거
- Free time 프리셋은 즉시 커밋 (3 row 일괄 시 일일이 저장 부담 회피)

### Coverage gate

기존: title 있으면 저장 가능. 신규: 모든 caseProducts.variantId가 schedule
items 중 한 row(committed)에 등장해야 활성화.
