## 2. ScheduleDocument 모바일 반응형

데스크탑 디자인이 모바일(375px)에서 양쪽 64px 패딩 → 콘텐츠 폭 247px로
좁아지는 구조. 가독성 떨어짐 + 시간 컬럼 너무 좁음.

### CSS 클래스 시스템

미디어 쿼리 타겟팅을 위해 주요 컨테이너에 클래스 추가:
- `sch-outer` — 최외곽 wrapper
- `sch-cover-inner` / `sch-day-inner` — 내부 패딩 컨테이너
- `sch-cover-title` / `sch-cover-date` — 커버 텍스트
- `sch-day-num` / `sch-day-date` — Day 헤더 숫자/날짜
- `sch-row` — 시간 + 콘텐츠 2열 그리드
- `sch-footer` — 하단 푸터
- `sch-tabs` — 그룹 탭 바
- `sch-flights-grid` / `sch-flight-detail` — 항공 섹션

## 3. 그룹 탭 Sticky

스케줄 페이지가 길어질 경우 그룹 탭이 위로 사라지면 그룹 전환 불편.
`position: sticky; top: 0; zIndex: 10` + `background: #e8e4de` (페이지 배경색)으로
스크롤 고정. 배경색 맞춰야 아래 콘텐츠가 탭 아래로 슬라이드되는 시각 이슈 없음.

## 4. 스케줄 블록 섹션 시각 계층

이전: 블록 섹션 헤더(Morning/Afternoon)와 항목 구분선 둘 다 `1px solid` 선 계열
→ 계층 구분 안 됨.

변경:
- 블록 헤더 하단 선: `1px solid #ede9e3` → `2px solid #d8d2ca` (굵게 + 짙게)
- 항목 사이 구분선: `1px solid #f5f2ee` → `none` (완전 제거)

결과: Morning/Afternoon 헤더가 섹션 시작점으로 명확히 읽힘.
항목들은 서로 붙어 있어 같은 블록 내 활동으로 묶인 느낌.

## 5. Invoice 표 모바일 컴팩트

데스크탑에서 `text-sm`인 표가 모바일에서 가로 스크롤 발생.
원인: `whitespace-nowrap` + `text-sm` 조합으로 행 폭 초과.

수정:
- 표 자체: `text-xs md:text-sm` (모바일 더 작게)
- 셀 패딩: 헤더/바디 `py-2 md:py-2.5 / py-2 md:py-3`
- 금액 셀만 `whitespace-nowrap` (숫자는 줄바꿈 안 되어야 하지만 설명 컬럼은 허용)
- Total 행: `text-sm md:text-base`

## 6. Quotation admin 이름 노출

Invoice(`/invoice/[slug]`)는 이미 `signer_snapshot`으로 admin 이름/직책 표시.
Quotation(`/quote/[slug]`)도 같은 포맷으로 admin 이름 + 회사명 노출 요청.

문제: `documents.created_by_admin_id`가 항상 null — `createDocument()`가 이 컬럼을
채우지 않음. Quotation은 agent review 플로우에서 생성되는 구조라 admin ID 없음.

해결: super admin을 회사 대표자로 fetch해서 fallback으로 사용.

```typescript
// Promise.all에 추가
supabase.from('admins').select('name, title').eq('is_super_admin', true).limit(1).maybeSingle()

// FK join 시도 (admins!documents_created_by_admin_id_fkey)
// TypeScript 추론이 배열 타입으로 잡아서 unknown 이중 캐스트 필요:
const creatorAdmin = (quote as unknown as { admins?: { name: string; title: string | null } | null }).admins
  ?? (superAdminRes.data as { name: string; title: string | null } | null)
  ?? null
```

`adminName = signer?.name ?? creatorAdmin?.name ?? null` — 우선순위:
1. `signer_snapshot` (finalize 시 동결된 admin)
2. FK join으로 가져온 `created_by_admin_id` admin
3. super admin fallback

신규 quotation도 동일하게 반영됨 (super admin이 회사 대표로 표시).
`signer_snapshot`이 있는 기존 finalize invoice는 그것이 우선.

## 7. Vercel 빌드 오류 2건

### 오류 1: duplicate className on flight detail paragraphs

`replace_all`로 `.sch-flight-detail` 클래스 추가 시, 이미 `className="sch-sans"` 있는
요소에 `className="sch-flight-detail"` 이 별도로 붙어서 duplicate className 발생.
JSX에서 같은 요소에 `className` prop이 2개 → React 경고 + 빌드 에러.

수정: `className="sch-sans sch-flight-detail"` (병합, 4곳).

### 오류 2: TypeScript 타입 충돌 on admins join

Supabase FK join `admins!documents_created_by_admin_id_fkey`의 추론 타입이
`{ name: string; title: string | null }[]` (배열)로 잡힘.
이를 `{ admins?: { name: string; title: string | null } | null }` (단일 객체)로
캐스트하려 하니 "neither type sufficiently overlaps" 오류.

수정: `unknown` 이중 캐스트 — `(quote as unknown as { admins?: ... })`.
`as unknown as T` 패턴은 타입 추론이 불가능한 Supabase join에서 일반적.

## 정성적 메모

- 모바일 대응을 CSS class + inline `<style>` 미디어 쿼리로 처리한 패턴은
  고객용 페이지(Tailwind 클라이언트 없음) 특성상 불가피. 데스크탑 렌더/인쇄는
  완전 무변경.
- `sch-row grid-template-columns: 64px 1fr`로 시간 컬럼 명시. 이전엔 데스크탑
  고정 80px라 모바일에서 콘텐츠 컬럼 너무 좁았음.
- Quotation admin 이름 — `created_by_admin_id`가 null인 구조적 이유는 agent review
  플로우가 admin identity 없이 document를 생성하기 때문. super admin fallback이
  현실적 해법. 향후 createDocument 시 admin ID를 저장하도록 변경해도 기존 quotation은
  super admin으로 표시되므로 무해.
- TypeScript `unknown` 이중 캐스트 — Supabase PostgREST join 타입 추론이 항상
  배열로 나오는 한계. 패턴 메모해 두면 유사 케이스에서 시간 절약.
- 블록 내 항목 구분선 제거가 오히려 깔끔함. 빽빽한 분리선이 도리어 시각 노이즈였음.
  헤더 선 굵기/색 강화로 계층이 더 명확해짐.
- D-1. 남은 것: 스케줄 시간 정렬 버그, internal notes 스타일, 컨텐츠 todo(계약서/설문/호텔 v19).
