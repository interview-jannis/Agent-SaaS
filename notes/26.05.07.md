## 6. Quotation admin 이름 노출

Invoice(`/invoice/[slug]`)는 이미 `signer_snapshot`으로 admin 이름/직책 표시.
Quotation(`/quote/[slug]`)도 같은 포맷으로 admin 이름 + 회사명 노출 요청.

문제: `documents.created_by_admin_id`가 항상 null — `createDocument()`가 이 컬럼을
채우지 않음. Quotation은 agent review 플로우에서 생성되는 구조라 admin ID 없음.

해결: super admin을 회사 대표자로 fetch해서 fallback으로 사용.

```typescript
// Promise.all에 추가
supabase.from('admins').select('name, title').eq('is_super_admin', true).limit(1).maybeSingle()

// FK join 시도 (admins!documents_created_by_admin_id_fkey)
// TypeScript 추론이 배열 타입으로 잡아서 unknown 이중 캐스트 필요:
const creatorAdmin = (quote as unknown as { admins?: { name: string; title: string | null } | null }).admins
  ?? (superAdminRes.data as { name: string; title: string | null } | null)
  ?? null
```

`adminName = signer?.name ?? creatorAdmin?.name ?? null` — 우선순위:
1. `signer_snapshot` (finalize 시 동결된 admin)
2. FK join으로 가져온 `created_by_admin_id` admin
3. super admin fallback

신규 quotation도 동일하게 반영됨 (super admin이 회사 대표로 표시).
`signer_snapshot`이 있는 기존 finalize invoice는 그것이 우선.

## 7. Vercel 빌드 오류 2건

### 오류 1: duplicate className on flight detail paragraphs

`replace_all`로 `.sch-flight-detail` 클래스 추가 시, 이미 `className="sch-sans"` 있는
요소에 `className="sch-flight-detail"` 이 별도로 붙어서 duplicate className 발생.
JSX에서 같은 요소에 `className` prop이 2개 → React 경고 + 빌드 에러.

수정: `className="sch-sans sch-flight-detail"` (병합, 4곳).

### 오류 2: TypeScript 타입 충돌 on admins join

Supabase FK join `admins!documents_created_by_admin_id_fkey`의 추론 타입이
`{ name: string; title: string | null }[]` (배열)로 잡힘.
이를 `{ admins?: { name: string; title: string | null } | null }` (단일 객체)로
캐스트하려 하니 "neither type sufficiently overlaps" 오류.

수정: `unknown` 이중 캐스트 — `(quote as unknown as { admins?: ... })`.
`as unknown as T` 패턴은 타입 추론이 불가능한 Supabase join에서 일반적.

## 10. Commission Invoice Mark Paid 통합

기존에 "Commission Invoice" 카드와 별개로 "Agent Settlement" 섹션이 있었는데,
흐름이 중복이고 UX가 복잡함. Commission Invoice 카드에 Mark Paid를 직접 달고
"Agent Settlement" 섹션 전체 제거.

### 변경 사항 (`src/components/CaseDocumentsSection.tsx`)

**Mark Paid 핸들러 (`doMarkPaid`)**:
commission_invoice 타입이면 결제 확인 후:
1. `settlements` 테이블에 신규 row 삽입 (settlement_number `#S-XXX` 자동 발번)
2. `cases.status` → `completed` (현재 `awaiting_settlement`인 경우에만)
3. agent에게 커미션 지급 알림 전송

```typescript
} else if (doc.type === 'commission_invoice') {
  const amount = doc.total_price ?? 0
  const { count } = await supabase.from('settlements')
    .select('*', { count: 'exact', head: true })
  const next = (count ?? 0) + 1
  const settlementNumber = `#S-${String(next).padStart(3, '0')}`
  await supabase.from('settlements').insert({
    settlement_number: settlementNumber,
    agent_id: agentId,
    case_id: caseId,
    amount,
    paid_at: paidAtValue,
  })
  await supabase.from('cases')
    .update({ status: 'completed' })
    .eq('id', caseId)
    .eq('status', 'awaiting_settlement')
  if (agentId)
    await notifyAgent(agentId,
      `${caseNumber} Commission paid (${docNumber}) — ${fmtUSD(amount / exchangeRate)}`,
      `/agent/cases/${caseId}`)
}
```

**`canEdit` 분기**: commission_invoice는 `actor === 'admin'` (발행자가 agent여도 admin만 결제 확인).
기존: `actor === doc.from_party`.

**`+ Add item` 제거**: commission_invoice와 deposit_invoice는 Add item 버튼 비노출.

**커미션 모달 금액**: KRW+USD → USD만. `return fmtUSD(krw / exchangeRate)`.

**Agent Settlement 섹션 전체 제거** (`src/app/admin/cases/[id]/page.tsx`):
- state: `agentSettlePaidAt`, `savingAgentSettle`, `agentSettleError` 제거

## 11. Pre-confirmation 상품 노출 정합성

### 문제

원래 설계: schedule 확정 전에는 원본 항목(original)만 보임. admin이 추가하거나 삭제한
변경사항은 확정 전까지 "미확정". 그러나 `removed_at`으로 soft delete된 원본 항목이
agent 쪽에서 사라져 버리는 문제.

원칙 재확인:
- 확정 전 삭제는 "삭제 요청" 상태 — 아직 확정 아님
- agent는 원본 항목 전체를 계속 봐야 함
- admin_added 항목만 확정 전 미노출

### 수정 (`src/components/SelectedProductsSection.tsx`)

```typescript
function activeItems(g: Group, showOriginalOnly = false): Item[] {
  if (showOriginalOnly) {
    // 확정 전: removed_at 무시하고 original 항목 전체 노출
    return (g.document_items ?? []).filter(it =>
      it.origin === 'original' || it.origin == null
    )
  }
  return (g.document_items ?? []).filter(it => !it.removed_at)
}
```

### 수정 (`src/app/agent/cases/[id]/page.tsx`)

`totalKrw` 계산 — pre-confirmation: removed_at 무시하고 original 항목 합산:

```typescript
const totalKrw = (!scheduleConfirmed && !quote?.finalized_at)
  ? (quote?.document_groups ?? [])
      .flatMap(g => (g.document_items ?? []))
      .filter(it => it.origin === 'original' || it.origin == null)
      .reduce((s, it) => s + (it.final_price ?? 0), 0)
  : (quote?.total_price ?? 0)
```
