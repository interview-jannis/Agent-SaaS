## §3. Partner Payouts 이중 곱셈 버그 수정

**파일**: `src/app/admin/cases/[id]/page.tsx`

### 문제
`document_items.base_price` = `baseUnitKrw × multiplier` (quotation 생성 시 총액으로 bake-in).  
기존 코드: `itemCost = base_price × item.quantity` → multiplier가 두 번 곱해져 ₩882M 오계산 (실제 ₩365M 수준 대비).

### 수정된 공식
```
base_price          → 일반 아이템 총비용 (이미 단가×수량 bake-in)
base_price / qty × scheduledDays  → Trip Services: 실제 일정 일수로 재계산
final_price         → 초과근무(overtime) 아이템
```

`contractedQty = item.quantity`, `scheduledDays` = Trip Services 그룹에서 variant별 실제 스케줄 일수.

### 파트너 결제 Reset 기능 추가
- `deletingPartner` state 추가 — 잘못 입력된 파트너 결제 레코드 삭제 후 재입력 가능
- Reset 버튼 → 파트너 이름 직접 입력 확인 (파괴적 액션 확인 UX 패턴 적용) → `supabase.from('partner_payments').delete().eq('id', existing.id)`

## §4. Partner Payouts 표시 개선

**파일**: `src/app/admin/cases/[id]/page.tsx`

### 금액 원화(KRW) 표시 통일
- "Paid $X of $Y" → "Paid ₩X of ₩Y"
- `fmtUSD(... / exchangeRate)` → `fmtKRW(totalPaid)` / `fmtKRW(totalSuggested)`

### 아이템 단가×수량 분해 표시
qty > 1 이면: `₩단가 × qty단위 = ₩합계` 형식으로 표시 (locked view + unpaid view 양쪽 적용)  
qty = 1 이면: `₩합계`만 표시

예시: `₩1,500,000 × 3d = ₩4,500,000`
