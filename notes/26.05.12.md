### 1. PowerShell 인코딩 손상 복구

이전 세션에서 `Set-Content` 무인코딩 사용으로 두 파일의 Unicode 문자가 깨짐.

**손상 패턴:**
- 3-byte UTF-8 문자 (예: `₩`, `▲`, `▼`, `→`, `✓`) → `??` (물음표 2개)
- 2-byte UTF-8 문자 (예: `·`, `×`) → 한글 문자 (`쨌` = `·`, `횞` = `×`)
- 닫는 태그 `</option>` 앞의 `—` → `??/option>` (`<` 포함 4바이트가 2개 `??`로)

**복구된 파일:**
- `src/components/SelectedProductsSection.tsx`: `₩`, `·`, `×`, `▲/▼` 복구
- `src/components/admin/ScheduleEditor.tsx`: `▼/▲`, `—`, `…`, `▾`, `→`, `·`, `✓`, `Café` 복구 (약 20곳)

**교훈:** PowerShell 파일 조작은 반드시 `[System.IO.File]::WriteAllText($path, $content, [System.Text.Encoding]::UTF8)` 사용. `Set-Content` 기본 인코딩은 UTF-16 LE(BOM) 또는 시스템 기본값으로 3-byte 이상 Unicode 손상시킴.

### 4. canEdit 권한 버그 픽스 (admin/cases/[id]/page.tsx)

**문제:** `isSuperAdmin || assigned_admin_id === currentAdminId` 조건으로 super admin 또는 해당 케이스의 담당 admin만 편집 가능. 다른 admin은 "Finalize Pricing & Issue Invoice" 등 모든 액션 비활성.

**원인 분석:**
- `assigned_admin_id`는 알림 라우팅(누구한테 noti 보낼지)에 사용되는 필드
- 실제 케이스 관리는 any admin이 할 수 있어야 함
- `assigned_admin_id`가 설정 안 된 케이스도 많음 (선택적 필드)

**수정:** `const canEdit = !!currentAdminId` — 로그인한 admin이면 누구나 편집 가능.

> 단, Settings/Contracts/Surveys/Admins 메뉴의 **수정** 권한은 super admin 전용 정책 유지 (CLAUDE.md 권한 분기 항목). canEdit 변경은 케이스 단위 작업에만 해당.

### 6. Health Screening 상품 분리 (upload-excel route.ts)

**배경:** K-Medical Health Screening 상품은 Female/Male/Junior가 검사 항목 구성 자체가 달라 description이 달라야 함. 기존엔 같은 `name = "VIP Premium"` 아래 variant_label로 묶여 description 공유.

**결정:** 별도 상품(product)으로 분리. variant 모델은 "같은 시술, 다른 횟수/등급" 용도 — 성별 구분은 별도 상품이 맞음.

**구현:** Excel 파일 구조는 그대로 두고 업로드 스크립트에 파싱 규칙 추가.
```ts
// src/app/api/admin/products/upload-excel/route.ts
if (normStr(r.subcategory) === 'Health Screening' && normStr(r.variant_label)) {
  r.name = `${normStr(r.name)} ${normStr(r.variant_label)}`  // "VIP Premium Male"
  r.variant_label = ''  // default/sole variant
}
```
- Excel: `name = "VIP Premium"`, `variant_label = "Male"` 유지
- DB: `name = "VIP Premium Male"`, variant_label = null (sole variant)
- 각 row의 description이 독립 product에 저장됨

**업로드 결과:** +10 new products (4센터 × Male/Female/Junior), 4개 구버전 "VIP Premium" → `deleteMissing=false`로 신규 추가 후 수동 삭제.

### 9. Vehicle per_day 자동 일수 동기 (pricing.ts + agent/product/page.tsx)

**문제 재발:** 날짜 미선택 시 차량 `2d`, 날짜 선택해도 `2d` 유지. 호텔(`per_night`)은 미선택 1n, 선택 시 자동 n박으로 정상 동작.

**원인:** `daysBetween(null, null)` → `nightsBetween(null, null) + 1` = `1 + 1 = 2`.  
호텔은 `nights` (live 변수) 를 직접 참조하지만 차량은 `it.days` (add 시점 저장값) 를 참조함.

**수정 — `src/lib/pricing.ts`:**
```ts
export function daysBetween(dateStart, dateEnd): number {
  if (!dateStart || !dateEnd) return 1   // ← early return 추가
  return nightsBetween(dateStart, dateEnd) + 1
}
```

**수정 — `src/app/agent/product/page.tsx`:**
```ts
const daysLive = useMemo(() => daysBetween(dateStart, dateEnd), [dateStart, dateEnd])
```
- `totalUSD`: `per_day` 아이템은 `it.days` 대신 `daysLive` 사용
- 칩 렌더: `isPerDay` 분기 추가 → `{daysLive}d (auto)` 표시, +/- 컨트롤 숨김
- 호텔 칩이 `{nights}n (auto)` 였던 것과 완전 동일 패턴으로 통일

### 10. Edit Selected Products UX — 자동스크롤 + sticky picker

**요청:** Edit 버튼 클릭 시 picker 영역으로 자동 스크롤. picker가 스크롤해도 항상 보이게.

**자동스크롤 — `src/app/admin/cases/[id]/page.tsx`:**
```tsx
const editPickerRef = useRef<HTMLDivElement>(null)
useEffect(() => {
  if (!editingProducts) return
  const id = requestAnimationFrame(() =>
    editPickerRef.current?.scrollIntoView({ behavior: 'smooth', block: 'end' })
  )
  return () => cancelAnimationFrame(id)
}, [editingProducts])
```
`requestAnimationFrame` — DOM 렌더 후 1프레임 뒤에 실행해야 실제 위치에 스크롤됨.

**Sticky picker:**
```tsx
<div ref={editPickerRef}
  className="sticky bottom-0 -mx-4 px-4 pb-4 pt-3 bg-white border-t-2 border-[#0f4c35]/20 rounded-b-2xl space-y-2">
  <div className="bg-green-50 rounded-xl border-2 border-[#0f4c35]/30 ...">
    Add to
    ...
  </div>
  {actionError && ...}
  <div className="flex items-center justify-end gap-2 ...">
    Cancel / Save
  </div>
</div>
```
- picker + actionError + Save/Cancel 세트를 sticky wrapper로 묶음
- 검색 드롭다운이 아래로 열리면 viewport 밖으로 숨어버리므로 `top-full mt-1` → `bottom-full mb-1` 변경

### 11. Completed 케이스 — Financials 섹션 최상단

**요청:** `status === 'completed'`일 때 Financials(견적/인보이스) 섹션을 Hero 직후 최상단으로.

**방법:** `space-y-5` → `flex flex-col gap-5`로 교체 후 CSS `order` 프로퍼티 활용.
- 취소 배너 / 미배정 배너 / Hero: `order-first` (= `order: -9999`)
- Financials: `completed` 시 `order: -1`, 아닐 때 `order: 0` (기본)
- 나머지 섹션: 명시적 order 없음 → `order: 0` → DOM 순서 유지

```tsx
// 외부 flex col
<div className="flex flex-col gap-5">
  <div className="order-first">취소 배너</div>
  <div className="order-first">Hero</div>
  <div className={status === 'completed' ? 'order-[-1]' : ''}>
    {/* Financials IIFE */}
  </div>
  {/* 나머지 섹션들 */}
</div>
```

### 12. Agent 비활성화 완전 차단 (3-layer)

**문제:** `agents.is_active = false`로 비활성화해도 이미 로그인한 agent가 모든 페이지에 접근 가능.

**수정 레이어 3개:**

#### Layer 1 — 로그인 차단 (`src/app/login/page.tsx`)
```ts
const { data: a } = await supabase
  .from('agents').select('id, onboarding_status, setup_completed_at, is_active')
  ...
if (a.is_active === false) {
  await supabase.auth.signOut()
  router.push('/deactivated')
  return
}
```

#### Layer 2 — 기존 세션 차단 (`src/components/AgentOnboardingGuard.tsx`)
```ts
.select('onboarding_status, setup_completed_at, is_active')
...
if (row?.is_active === false) {
  router.replace('/deactivated')
  return
}
```
`AgentOnboardingGuard`는 `/agent/*` 모든 레이아웃에 마운트돼 있어 탭 이동 시마다 재검사.

#### Layer 3 — 전용 차단 페이지 (`src/app/deactivated/page.tsx`, 신규)
- Super admin 이메일을 DB에서 실시간 조회 (`admins WHERE is_super_admin = true`)
