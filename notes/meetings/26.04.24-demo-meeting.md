## ⭐ 핵심 결정

### 1. 결제 ↔ 스케줄 흐름 변경 (B로 재구성)

**이전 흐름 (A)**: 견적 → **결제** → 스케줄 → 여행 → 정산
**새 흐름 (B)**: 견적 → 스케줄 → **결제** → 여행 → 정산

#### 이유
- VIP 의료관광 업계 관습: 고객이 일정·호텔·세부사항 확정한 뒤 결제
- "Quotation"(견적) vs "Invoice"(확정 거래 청구) 명칭 정합성
- MVP 출시 = 실제 고객 유치 목적이라 비즈니스 표준에 맞춰야 함

#### 작업 범위
- DB: `case_status` CHECK 재정의 + `quote_sent` 신규 상태 추가
- Status 전환 로직 전부 재배열 (견적 생성 시점 / 스케줄 업로드 가능 시점 / 결제 확정 시점)
- UI 라벨 (Quotation/Invoice 분기), Action Required 정렬, Pipeline 순서, 알림 트리거 시점
- 영향 파일 14개 추정 (admin/agent 양쪽)

→ **5/8 마감 안에 진행**, 이번 주~다음 주 초 작업.
