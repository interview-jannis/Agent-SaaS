## 결정 사항

### 1. 브랜드명 변경: Tiktak → TikkTakk
- K를 한 개씩 더 붙임 (TIKK + TAKK)
- 의미: 시계 초침 소리 → "시간을 정밀하게 설계해서 VIP 여정을 완성한다"
- 코드베이스 전역 텍스트 치환 필요

### 2. Agent 온보딩 사인 주체
- NDA / Partnership 계약서 admin 카운터사인 → **대표님**이 직접 서명

### 3. 온보딩 링크 발송 방식
- 현재: invite 링크 1개만 발송
- 변경: **2개 링크** 함께 발송
  1. 온보딩 링크 (`/invite/[token]`)
  2. SaaS 앱 메인 링크 (온보딩 완료 후 로그인용)
- 이유: 온보딩 완료 후 invite 링크로는 앱 재진입 불가능

### 4. 계약 문서 — 홍이사님 전달 예정
- NDA 본문
- Partnership 본문

### 5. Agent > Home 탭 → 절차 소개 화면으로 전면 개편
현재 홈 탭에 상품 목록이 바로 노출 → **9단계 비즈니스 플로우 소개** 화면으로 교체

소개 순서:
1. 에이전트 & Tiktak(admin) 소개
2. 비즈니스 가치 / 차별성 설명
3. 원하는 일정에 맞춰 상품을 선택할 수 있다
4. 상품 선택 후 제안서(Quotation)가 고객에게 발송된다
5. 제안서 컨펌 후 계약이 진행된다
6. 계약 후 디파짓 50% 입금이 필요하다
7. 디파짓 입금 후 상세 스케줄이 나온다
8. 출발 하루 전까지 잔금 50% 입금
9. 이후 프리미엄 K-여행을 즐기시면 된다

> Agent가 고객과 대면 상담 시 이 화면을 보여주며 절차를 설명하는 용도.
> 설명 끝나고 "상품 보시겠습니까?" → Product 탭으로 이동.

### 6. Agent > Home 탭 → "Product" 탭으로 이름 변경
- 기존 Home 탭의 상품 카탈로그 영역을 별도 **Product** 탭으로 분리
- 탭 순서: Home(절차 소개) → **Product** → Cases → Clients → Payouts → Dashboard → Profile

### 7. K-Medical 상품 순서 재정의
1. Health Screening (건강검진)
2. Dental (치과)
3. Eye (안과)
4. Women's Health (여성병원)
5. Korean Medical (한방) — 데이터 추가 후 마지막

### 8. Female / Male 상품 — 복수 선택 불가
- 현재: 같은 그룹에서 Female 건강검진 + Male 건강검진 동시 선택 가능
- 변경: **한 그룹 = 1인 기준** → Female 또는 Male 중 하나만 선택 가능
- 여러 성별 구성원이 있는 경우 그룹을 분리해서 각각 선택

### 9. 인당 견적서
- 현재: 그룹 합산 견적서 1개만 발행
- 추가 필요: **개인별(인당) 견적서** 각각 출력 가능하도록
- 최종 발행은 합산 1개 + 개인별 서브 첨부 형태

### 10. 통역 상품 — 시간 옵션 확장
- 현재: 고정 단위
- 변경: 시간 단위를 늘릴 수 있도록 (예: 4시간 / 8시간 / 연장 옵션)

### 11. Agent > Product 글자 크기 키우기
- 상품 카드의 상품명 텍스트가 너무 작음 → 한 단계 키울 것

### 12. Cases 생성 직후 Financials 상단 배치
- 현재: Financials 섹션이 하단에 위치
- 변경: 계약 체결 전 (`awaiting_contract` 단계) → **Financials 섹션을 상단에 노출**
- 이유: 에이전트가 고객에게 빠르게 Quotation 링크를 복사해서 보낼 수 있어야 함

### 13. Quotation / Invoice — To. 표시
- 현재: To 란에 그룹명 또는 클라이언트 이름이 어떻게 표시되는지 확인 필요
- 그룹명 / 대표 클라이언트 이름 중 적합한 표시 결정 필요

### 14. Agent → Client Invoice 발행 불필요
- 기존 플로우에서 Agent → Client 방향 Invoice 발행이 있었으나 **불필요**
- 에이전트가 고객한테 받는 건 영수증/입금 확인 개념 (별도 확인 필요)
- **변경**: Invoice 발행은 Admin → Agent 방향만 (기존 Deposit / Final / Commission)

### 15. Finalize Pricing 금액 = 파트너사 원가 기준으로 표시
- 현재: 마진 적용가로 표시됨
- 변경: Admin이 Finalize 시 보는 금액 → **파트너사에 지불할 원가 금액**으로 표시
- 고객/에이전트 방향 Invoice에는 마진 적용된 달러가로 나가는 것은 동일

### 16. 환율 변경 시 기존 문서 금액 잠금
- 현재: Settings에서 환율 수정 시 기존 견적/Invoice의 달러 금액도 자동 재계산됨
- 변경: **한번 발행된 문서(Quotation/Invoice)는 금액 수정 불가** — 완전 잠금
- 환율은 신규 견적 생성에만 영향

### 17. 상품 등록 가격 고정 — 1500원 기준 달러 환산
- 상품 원가는 KRW로 등록, **1500원 고정 환율**로 달러 변환한 값이 판매 달러가
- 실제 환율이 변동해도 상품 달러 가격은 바뀌지 않음
- 환율 리스크는 회사가 갖고 감 (환율 하락 시 원가 부담 증가, 상승 시 이익 증가)
- 설정값이 아니라 코드 고정값으로 처리

### 18. Final Invoice 이후 파일 업로드 기능 추가
- 여행 전(잔금 확정 후 ~ 출발 전) 단계에 파일 업로드 섹션 추가
- 업로드 대상:
  - 보험 증권
  - 여행 주의사항
  - 공항 픽업 안내 (몇 터미널, 몇 번 기둥 등)
  - 기타 여행 안내 서류

## 정성적 메모

- 시뮬을 직접 A→Z 실행해보니 전체 흐름은 작동함. 상품 데이터 품질이 관건.
- "상품이 제일 중요한데" — 정리만 되면 엑셀 업로드로 바로 반영 가능
- 환율 고정 정책: 1500원 기준 판가 세팅 → 실환율 1400원대이면 100원/달러 이익
  환율 크게 하락 시 판가 재조정 필요하지만 당분간 유지
- 사진 등록 방법 고민: 현재 건 바이 건 방식 → 폴더 기반 일괄 등록 방법 추후 검토
- 그룹 구성 복잡도(가족 4명, 성별/나이 다름) → 인당 케이스 선택 + 합산 모델이 현실적
- 화요일부터 시뮬레이션 시작 목표
