## 데이터 가공 — 상품 등록 포맷

### 등록 권한
- **일반 admin도 등록 가능** (super admin 한정 X) — "바쁠 때 다 같이 올려야 함"

### 업로드 방식 (구현 필요)
- 현재: 사이트에서 1건씩 등록만 가능
- **추가 필요**: **엑셀 일괄 업로드 기능**
  - 정해진 포맷 엑셀 파일 업로드 → 자동 분류 → admin 상품 등록 + agent쪽 자동 매칭
  - 카테고리별 자동 분류
  - "사진만 사람이 직접 올리면 됨"

### 사진
- 대학병원, 클리닉 등 파트너 홍보 자료에서 사진 가져옴 (기관 브랜드 일관성)
- 엑셀 업로드 후 사진은 사이트에서 추가

### 카테고리별 가격 정책

| 카테고리 | 정책 |
|---|---|
| Medical (대학병원 검진 등) | 오픈 가격 그대로. "가격 속이면 안 됨". 컨시어지 50% 추가 — 사실상 패키지 |
| Beauty (피부과/시술) | 오픈 가격 그대로 |
| Wellness (투어/스파/액티비티) | 원가 그대로 (마진 거의 X) |
| Wellness 중 **스파 조합 패키지** | 2~3개 묶어서 4~5시간짜리 만들고 뻥튀기 (ex: 4~500만원) |
| Hotel | 마진 **20~30%** ❓ ← **대표님 의사결정 대기** |
| Subpackage (통역/컨시어지/의전/경호) | **50%** ❓ ← **대표님 의사결정 대기** |
| 보험 | 홍이사님 별도 검토 |

### 스파/일반 상품 처리 노트
- 100만원 이하 일반 상품들은 그냥 등록만, 견적에서 추가만 가능
- 단가 못 올리는 항목은 마진 의미 없음 → "패키지 조합으로 가격 형성"

## 4가지 계약서 (모두 홍이사님 검토 예정)

1. **NDA 계약서** — 기존
2. **Partnership 계약서** — 기존
3. **에이전트 ↔ 고객 계약서** — 신규
4. **에이전트 ↔ 고객 ↔ 어드민 3중 계약서** — 신규 ⭐ "사업에서 제일 중요한 부분"

### 3중 계약서 핵심 조항 (방향성)
- **결제 조건**: 50% 디파짓 + 50% 비행기 타기 전 ❓ ← **대표님 의사결정 대기**
- **컨시어지/보험/의전/안전 책임 모두 어드민이 짐**
- **3자 모두 사인** (고객, 에이전트, 어드민)
- **환불 정책**: 원칙적으로 환불 없음
  - 이유: 파트너 기관 취소 수수료 + 컨시어지 비용으로 상쇄됨

### 계약서 사인 흐름 (현재 — 강화 필요)
- 현재: agent만 NDA/Partnership 사인
- **필요**: admin도 사인 → agent가 admin 사인까지 열람 가능

## 전체 SOP (A→Z)

### Phase 1: 영업 (오프 사이트)
1. 에이전트가 VIP 고객에게 **소개서/제안서 자료**로 영업
2. VIP 관심 표명 → 만남 (대면)

### Phase 2: 상품 설명 → 견적 (SaaS)
3. 에이전트가 SaaS 오픈해서 상품 설명
   - **상품 + 기관 정보 함께** (WHY/WHAT/WHO)
4. VIP 상품 선택
5. 견적서(Quotation) 생성 → 고객에게 **링크** 전달
   - 견적서 발송 시 **고객 정보 등록 필수**

### Phase 3: 계약 → 결제
6. 고객이 견적 수락
7. **에이전트 ↔ 고객 ↔ 어드민 3자 계약 체결** (3장 사인)
8. 에이전트가 고객한테 디파짓 송금 받음
9. **어드민이 에이전트에게 디파짓 invoice 발송**
10. 에이전트가 어드민에게 디파짓 송금
    - 직접 송금이 아니라 에이전트 거치는 이유: "에이전트도 매출이 필요" + 신뢰 비즈니스 ← **대표님 의사결정 대기**

### Phase 4: 스케줄
11. 어드민이 한국 파트너 기관에 일정 잡으며 **스케줄 생성**
    - **스케줄은 무조건 링크로 조회** (PDF 첨부 X)
    - **엑셀 업로드 → 자동 링크 생성** (quotation/invoice와 같은 패턴) ← 구현 검토
12. 여행 직전: **잔금 50% invoice 발행**
    - 이때 invoice에 **법인 도장이 들어가야 함** (이미지 들어갈 수 있는지 확인 필요)

### Phase 5: 여행
13. 여행 중 풀 패키지 컨시어지 제공
14. **추가 일정 발생 시 추가 invoice 발행** (예: 일정 변경 → 비행기 일정 조정 + 새 스케줄 + 추가 견적)
15. 여행 완료 → 귀국

### Phase 6: 후기 → 정산
16. **설문조사** (10~15개 객관식 + 마지막 1개 주관식)
    - "한국 여행 어떠셨습니까?", "컨시어지는 어땠습니까?", "차량은?" 등
    - 마지막: "보완할 점 메시지 남겨주세요"
17. **에이전트가 어드민에게 커미션 invoice 업로드** → 어드민 송금
18. 끝

## Invoice 종류 (4가지)

1. **디파짓 invoice** — 계약 후, 에이전트 → 고객
2. **잔금 invoice** — 여행 직전, 에이전트 → 고객 (도장 들어감)
3. **추가 invoice** — 여행 중 추가 일정 발생 시
4. **커미션 invoice** — 여행 후, 에이전트 → 어드민

케이스에 따라 일부만 발생할 수 있음.

## 스케줄 자동 생성 (구현 검토)

기존: 어드민이 수기로 PDF 만들어 업로드 → 링크 생성

**개선 방향**: 엑셀에 데이별로 정리(예: Day1 / Day2 / 몇 시 ~ 몇 시 / 기관) → 엑셀 업로드 → 웹에서 자동 테이블 → 링크 생성

이미 invoice가 데이터 채워서 자동 생성되는 패턴 → 같은 패턴 적용 가능. 다음주 중 트라이.

장기적으로는 AI가 자동 스케줄 생성까지.
