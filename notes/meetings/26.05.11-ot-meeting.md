## 비즈니스 플로우 요약 (이태호 부사장님 설명)

### 핵심 원칙
- **Deposit 정책 엄수**: 50% 디파짓 입금 전까지 어떤 진행도 없음 (스케줄 작업, 에이전트 모두)
- **후기 수집**: 여행 종료 후 에이전트가 클라이언트 후기를 받아 제출 → 서비스 품질 개선 루프
- **커미션 지급**: 후기 제출 완료 → 에이전트 커미션 인보이스 발행 → 정산. 여기까지가 하나의 여정 완성

### 리스크 관리
- SaaS 가용성: 네트워크 불안정 등 에러 가능성 → 어드민 24시간 모니터링 체계 필요
- 개인정보 보호: NDA 계약서에서 고객 정보 관리 항목 엄격하게 강조
- 컨시어지: 한국 현지에서 고객을 전담할 파트너 확보가 컴플레인 최소화의 핵심 — 별도 파트너 발굴 필요

## 데모 주요 내용

### Admin 화면
- **Overview**: 기간별 케이스 수, 매출, 커미션 실적 확인
- **Cases**: 상태별 케이스 파이프라인. 현재 비어 있는 상태로 시연
- **Products**: 현재 상품 데이터가 너무 산만함 → 이번 주 내 정리 필요
  - 이미지는 AI 생성 임시 이미지 (실제 이미지 교체 필요)
  - 좌측 금액 = 파트너 원가, 우측 금액 = 마진 반영 고객가. **어드민 화면을 에이전트에게 절대 노출 금지**
- **Agents**: 계약 완료 에이전트 + 온보딩 중 에이전트 구분
- **Clients**: 에이전트가 등록한 고객 전체 조회 (어드민은 전체, 에이전트는 본인 것만)
- **Settlement**: 파트너사 정산 + 에이전트 커미션 정산 탭 분리
- **Contracts**: NDA·Partnership 계약서 관리. Super Admin만 수정 가능
- **Surveys**: 여행 후기 질문 편집 및 응답 조회
- **Audit Log**: 모든 변경 이력 타임스탬프 기록
- **Settings**: 환율·회사마진·은행계좌 관리

### Agent 온보딩 (라이브 시연)
- 어드민 → Invite Agent → 링크 생성 → 최진영 메일 발송 → 클릭하여 온보딩 진입
- Orientation 확인 → NDA(이름·국적 입력·서명) → Partnership(동일) → 어드민 알림
- 어드민에서 카운터사인 후 Approve → 에이전트 계정 설정(이메일/비밀번호/은행계좌)
- 에이전트 화면 진입

### Agent 화면
- **Home**: 클라이언트 소개용 마케팅 페이지. Why TikkTakk, How It Works 6단계
- **Product**: 상품 카탈로그. 카테고리 필터, 그룹 구성, 상품 담기, Create Quote
- **Cases**: 본인 케이스만 표시. 상태별 파이프라인
- **Clients**: 본인 등록 고객만 표시
- **Payouts**: 커미션 정산 내역
- **Dashboard**: 개인 성과 통계

### 케이스 생성 시연 (4인 가족 예시)
1. Product 탭 → 상품 선택 (K-Medical, K-Starcation 등)
2. 그룹 구성: 엄마/아빠/자녀1·2 개별 + Shared Activity (전원 공통)
3. Create Quote → 고객 미등록 시 고객 등록 → 케이스 생성
4. 어드민에 알림 → Awaiting Contract
5. 3자 계약서 사인 (에이전트 + 클라이언트 + 어드민) → Awaiting Deposit
6. 디파짓 입금 확인 → Awaiting Schedule
7. 어드민 스케줄 에디터: 상품별 날짜·시간·담당 컨시어지 배정 → 저장 → Reviewing Schedule
8. 에이전트 Confirm → Awaiting Pricing
9. 어드민 가격 Finalize → 잔금 인보이스 발행 → Awaiting Payment
10. 잔금 확인 → Awaiting Travel → 여행 → Mark Travel Complete
11. 후기 제출 → Awaiting Settlement → 커미션 인보이스 → Mark Paid → Completed

## 결정 사항 및 이슈

### 1. 상품 데이터 정리 (이번 주 완료 목표)
- 현재 100여 개 → 카테고리별 파트너 2~3개로 압축, 최고급 위주
- **K-Medical**: 성인남녀·유아 건강검진 2~3개 프로그램으로 재구성, 국제 의료센터 가격 기준
- **K-Beauty**: 클리닉 실명 노출 여부 논의 필요 → 이번 주 내 의견 취합 (이사님들)
- **K-Education**: 기획 필요. 아직 내부 방향 미정 — 이번 주 아이디어 수집
- **K-Starcation**: 케이팝 오디션 등 콘텐츠 보완 필요
- **Subpackage**: 컨시어지·호텔·차량·통역·경호의전 단가 입력 필요

### 2. 법적·규제 검토 필요 (이승기 이사님 제기) — **긴급**
- 한국 전자상거래법 / 개인정보보호법 준수 여부 확인
- 사우디·두바이 온라인 판매 및 고객 정보 처리 규제 확인
- 병원 정보 공개 관련 의료법 검토
- 환율 고정 적용 방식의 금융 규제 여부 확인
- → **이태호 부사장님 + 이승기 이사님이 각각 파트 나눠 조사**

### 3. 브랜드명 상표 문제 (기존 이슈 재확인)
- TikTok 유사 상표 문제 → TIKK / TikkTakk 가능 여부 변리사 확인 필요
- 공식 런칭 전까지는 외부 공개 최소화

### 4. 마진 구조 재확인 및 최종 확정 필요
- 현재 세팅: 고객가 = 원가 × (1 + 50% 회사마진) × (1 + 에이전트 커미션)
- 에이전트 커미션: 15% → 20% → 25% (월 완료 환자 수 기준)
- 최대 고객 부담: 원가 × 1.5 × 1.25 = 원가의 약 187.5% (65% 마진 관점 → 최대 75%)
- **대표님과 함께 최종 수치 컨펌 필요** — 지금 세팅된 수치는 잠정치

### 5. 환율 정책
- 현재 환율: 실시간 반영 아님, 어드민에서 수동 업데이트
- 현재 설정: 1500 KRW/USD (시장 1450~1460 기준, ~3~5% 버퍼)
- 환차익/환차손 관리 방법 정립 필요

### 6. 환불 정책
- 여행 출발 3일 전까지 취소 시 환불 가능 (잠정 방향)
- 스케줄 확정 후 원가 지출된 경우 부분 환불 처리 기준 필요
- → 3자 계약서에 환불 조항 명시 예정 (야놀자 정책 참고)

### 7. 포인트 제도
- VIP 대상 사업에서 포인트 제도는 불필요 → 도입 안 함

### 8. 고객 링크 공유 방식
- 현재: 복사 후 수동 발송
- 요청: WhatsApp·카카오 등 메신저 직접 공유 버튼 추가
- WhatsApp Business 계정 인증 진행 중 (메타 인증 지연) → 추후 기능으로 추가
