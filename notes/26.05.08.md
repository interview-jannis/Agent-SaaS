### 수정 내용 (`data/scripts/generate_v21.js`)

## 2. Agent Home — "Free" vs "Price on request" 구분

### 배경

Subpackage margin 설정이 admin settings에서 disabled일 때 모든 Subpackage 상품 가격이 ₩0이 됨 (무상 제공 정책). 이전에 `$0.00` 표시 → amber `Price on request`로 개선했으나, Sofitel ROYAL/Noble Klasse처럼 **진짜 가격 미설정** 상품과 **의도적 0원(무상)** Subpackage 상품이 같은 표시를 쓰는 문제 발생.

### 구분 로직

| 상황 | 표시 |
|------|------|
| `isSubpkg && subpackageMult(config) === 0` | **Free** (gray, 무상 제공) |
| `min === 0 && max === 0` (위 조건 외) | **Price on request** (amber, 가격 문의) |
| 일반 가격 | `$X,XXX.XX` |

### 수정 위치 (`src/app/agent/home/page.tsx`)

**1. `priceLabel()` 함수** (카드 가격 표시):
```typescript
if (isSubpkg && subpackageMult(subpkgConfig) === 0) return 'Free'
if (min === 0 && max === 0) return 'Price on request'
```

**2. `renderVariantRow` 가격 표시** (variant picker 버튼):
```tsx
{usd === 0 && detailIsSubpkg && subpackageMult(subpkgConfig) === 0
  ? <p className="text-sm font-semibold text-gray-500">Free</p>
  : usd === 0
    ? <p className="text-sm font-bold text-amber-600">Price on request</p>
    : <p className={`text-sm font-bold ${inCart ? 'text-[#0f4c35]' : 'text-gray-900'}`}>
        ${usd.toLocaleString('en-US', { minimumFractionDigits: 2, maximumFractionDigits: 2 })}
      </p>
}
```

**3. 2-level 아코디언 그룹 range 표시**:
```typescript
const range = (detailIsSubpkg && subpackageMult(subpkgConfig) === 0)
  ? 'Free'
  : (min === 0 && max === 0)
    ? 'Price on request'
    : min === max
      ? `$${min.toLocaleString('en-US', { maximumFractionDigits: 0 })}`
      : `$${min.toLocaleString('en-US', { maximumFractionDigits: 0 })} – $${max.toLocaleString('en-US', { maximumFractionDigits: 0 })}`
```

단일 variant 모달 헤더 (`toUSD()`)는 `priceLabel()`을 호출하므로 자동 반영.

## 4. 가이드 콘텐츠 편집 기능 (슈퍼 어드민 전용)

### 배경

`/admin/guide` 화면은 역할별 Tiktak 사용 설명서인데, 스크린샷·설명 텍스트가 전부 하드코딩.
실제 운영 중 화면이 바뀌면 코드를 다시 배포해야 해서 슈퍼 어드민이 직접 업데이트할 수 있어야 함.
일반 어드민은 조회만 가능, 슈퍼 어드민만 편집 가능.

### 데이터 저장 방식

`system_settings` 테이블 기존 key/JSONB value 패턴 활용. 신규 key: `guide_content`.

```json
{
  "screenshots": { "overview": "https://...", "case_admin_awaiting_contract": "https://..." },
  "descs":       { "overview": "커스텀 설명 텍스트", "agent_home": "..." },
  "actions":     { "admin_awaiting_contract": "...", "agent_awaiting_settlement": "..." }
}
```

- `screenshots` 키: 섹션 key(e.g. `overview`, `agent_home`) 또는 `case_{admin|agent}_{status}`
- `descs` 키: 섹션 key → 커스텀 설명
- `actions` 키: `{admin|agent|client}_{status}` → 케이스 파이프라인 액션 텍스트
- SQL 없음 — 기존 테이블 재사용, upsert(`onConflict: 'key'`)로 저장

### Supabase Storage

버킷: `guide` (public). 경로: `screenshots/guide-{uploadKey}.{ext}`
- 업로드 시 `upsert: true` → 같은 경로에 덮어쓰기 가능
- 업로드 완료 후 `getPublicUrl()`로 URL을 자동으로 URL 필드에 채움

### `src/components/GuideContent.tsx` 주요 변경

**신규 타입/상수**
```typescript
type GuideEdits = {
  screenshots: Record<string, string>
  descs:       Record<string, string>
  actions:     Record<string, string>
}
const EMPTY_EDITS: GuideEdits = { screenshots: {}, descs: {}, actions: {} }
```

**신규 컴포넌트**
- `ImageUploadField` — URL 텍스트 입력 + 파일 업로드 버튼(hidden `<input type="file">` trigger), Storage upsert 후 URL 자동 반영
- `EditTextarea` — 설명 텍스트 인라인 편집
- `BrowserFrame` — 스크린샷 브라우저 프레임 래퍼. `onError={() => setFailed(true)}`로 404 이미지 silently 숨김

**슈퍼 어드민 감지 및 편집 모드**
```typescript
const { data: adminRow } = await supabase
  .from('admins').select('is_super_admin').eq('auth_user_id', user.id).maybeSingle()
if (adminRow?.is_super_admin) setIsSuperAdmin(true)
```
- 마운트 시 `system_settings.guide_content` fetch → `edits`/`savedEdits` state 초기화
- 헤더 우상단 "콘텐츠 편집" 버튼 (슈퍼 어드민 + 비편집 상태일 때만 노출)
- 편집 모드 진입 시 초록색 pulse 배너 + Save/Cancel 버튼
- Save: `system_settings` upsert → `savedEdits` 업데이트 → 편집 모드 종료
- Cancel: `edits`를 `savedEdits`로 롤백 → 편집 모드 종료

**SectionCard·CasesFlow 편집 UI**
각 섹션 카드/케이스 파이프라인 스텝을 열면 편집 모드 시 초록 배경 편집 패널 노출:
- 설명 텍스트 textarea
- 스크린샷 URL 입력 + 업로드 버튼 + 미리보기

### `src/components/AgentGuideContent.tsx` 주요 변경

에이전트 가이드 페이지도 동일한 `guide_content` key를 읽어 overrides 반영.

**마운트 시 fetch**

## 5. 상품 이미지 Bulk Upload

### 파일명 규칙

- `P-001.jpg` → 상품 #P-001 primary 이미지
- `P-001-2.jpg` → 상품 #P-001 추가 이미지 (order 2)
- `P-001-3.png` → 상품 #P-001 추가 이미지 (order 3)

대소문자 무관.

### API (`src/app/api/admin/products/bulk-images/route.ts`)

파일명 파싱 정규식: `/^p-(\d+)(?:-(\d+))?\.([a-z0-9]+)$/i`

동작:
1. 매칭된 상품의 기존 `product_images` rows + Storage 파일 삭제
2. 새 파일 업로드 (`product-images` 버킷)
3. `product_images` insert (is_primary, order 컬럼)

응답: `{ matchedProducts, totalImages, failed, unmatched, failedFiles, results }`

매칭 안 된 파일명(상품 없거나 파일명 형식 오류) → `unmatched` 배열에 기록.

### UI (`src/app/admin/products/page.tsx`)

"Upload Excel" 버튼 앞에 "Bulk Images" 버튼 추가. 클릭 시 모달:
- 다중 파일 선택 input
- 선택 파일 목록 미리보기
- Upload 버튼 → 결과 요약 (matched / unmatched / failed 상세)

## 11. 발행 문서 금액 잠금 + 상품 달러가 1500원 고정환산

### 배경

5/8 미팅 결정:
- 한 번 발행된 문서(Quotation/Invoice)는 환율 변경 후에도 금액 고정
- 상품 달러가는 Settings 환율과 무관하게 1500원 기준으로 고정 환산

### 변경 내용 (`src/lib/documents.ts`)

`documents` 테이블에 `price_rate_snapshot` 컬럼 추가 (NUMBER, nullable).
문서 생성 시 현재 환율(기본 1500)을 스냅샷으로 저장.
`QuoteDocument`는 `price_rate_snapshot` 값으로만 KRW→USD 변환 → 이후 환율 설정 변경이 기존 문서에 영향을 주지 않음.
`null`이면 1500 fallback.

### 마이그레이션

`documents.price_rate_snapshot` 컬럼 추가 필요 (Supabase 대시보드에서 직접 추가 or SQL).

```sql
ALTER TABLE documents ADD COLUMN IF NOT EXISTS price_rate_snapshot numeric DEFAULT 1500;
```
